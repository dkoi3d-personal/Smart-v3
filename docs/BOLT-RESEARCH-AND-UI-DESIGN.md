### What Makes Bolt.new UX Good

1. **Chat-First Interface** - Conversational input at the bottom, not hidden in a form
2. **Immediate Visual Feedback** - See your app as it builds
3. **Persistent Context** - Iteratively refine with follow-up prompts
4. **File Tree Visibility** - Always see what's being created
5. **Terminal Transparency** - Watch npm install, builds in real-time
6. **Minimal Friction** - No sign-up wall for trying it

## Competitive Analysis

### Bolt.new's Weaknesses

| Weakness | Impact |
|----------|--------|
| **31% success rate** on complex features | Falls apart past 15-20 components |
| **No memory** between sessions | Re-explain everything every time |
| **Token-hungry** | Debugging loops burn through budget |
| **Generic** | No domain expertise, no compliance |
| **Cloud-only** | Non-starter for hospitals |
| **Single agent** | Can't parallelize complex work |
| **No testing** | Generates code, crosses fingers |
| **No collaboration** | Solo dev tool only |
